# Product Backlog: Rainbow Roots

This backlog organizes the required features into Epics and User Stories to provide a clear development roadmap.

---

### Epic: User & System Foundation

*This epic covers the core authentication, authorization, and administrative functionalities.*

-   **User Story**: As a System Admin, I want to be able to create, edit, and deactivate user accounts (for Teachers and Parents) so that I can manage system access.
-   **User Story**: As a User (Teacher, Parent), I want to be able to log in to the system securely with my email and password.
-   **User Story**: As a User, I want to be able to log out of the system.
-   **User Story**: As a User, I want to be able to reset my password if I forget it.
-   **User Story**: As a System Admin, I want to be able to link a Parent account to one or more Student profiles.

---

### Epic: Student Profile & Assessment Management

*This epic focuses on the creation and maintenance of student records.*

-   **User Story**: As a Teacher, I want to create a detailed profile for a new student, including their name, date of birth, and other relevant personal details.
-   **User Story**: As a Teacher, I want to record the results of an initial assessment for a student to establish a baseline.
-   **User Story**: As a Teacher, I want to be able to view a list of all students assigned to me.
-   **User Story**: As a Teacher, I want to be able to edit a student's profile information.

---

### Epic: Learning Path Management

*This epic covers the creation and assignment of educational paths.*

-   **User Story**: As a Teacher/Admin, I want to create and manage a library of pre-defined learning paths.
-   **User Story**: As a Teacher/Admin, I want to define the stages, lessons, and materials within each learning path.
-   **User Story**: As a Teacher, I want to assign a specific learning path to a student based on their assessment.
-   **User Story**: As a Teacher, I want to be able to change or modify the learning path assigned to a student at any time.

---

### Epic: Daily Progress Tracking

*This epic focuses on the teacher's daily interaction with the system.*

-   **User Story**: As a Teacher, I want to see the scheduled lesson for a student for the current day based on their assigned path.
-   **User Story**: As a Teacher, during or after a lesson, I want to add a detailed note about the student's performance, challenges, and achievements.
-   **User Story**: As a Teacher, I want to be able to upload a document, image, or video as evidence of the student's progress during a session.
-   **User Story**: As a Teacher, I want to view a chronological history of all notes and uploads for a specific student.

---

### Epic: Reporting & Analytics

*This epic covers the generation of progress reports.*

-   **User Story**: As a Teacher, I want to generate a comprehensive progress report for a student for a given date range.
-   **User Story**: As a Teacher, I want the report to include a summary of progress, all notes, and links to the evidence files from that period.
-   **User Story**: As a Teacher, I want to be able to print the report or save it as a PDF.

---

### Epic: Parent Portal

*This epic defines the experience for the parents.*

-   **User Story**: As a Parent, I want to log in and see a dashboard for my child (or children).
-   **User Story**: As a Parent, I want to be able to view all the progress notes and evidence that the teacher has added for my child.
-   **User Story**: As a Parent, I want to see the learning path my child is on and the materials associated with their current lessons.
-   **User Story**: As a Parent, I want to view any homework or activities assigned to my child.
